API endpoints
POST/api/triage/extract-id
Accepts multipart image upload via Multer → reads file as base64 → calls Gemini Vision with strict JSON schema → returns parsed patient fields. On OCR failure: returns isPartial:true with whatever fields were confidently parsed, so the frontend pre-fills the editable form.
POST/api/triage/analyze-symptoms
Accepts { messages: [{role, content}] } conversation log → sends full history to Gemini with system prompt enforcing JSON output: { department, priorityTier, reasoning, followUpQuestion }. Multi-turn: if Gemini returns a followUpQuestion, relay it to the user. On AI failure: return GREEN priority + General Medicine as safe fallback.
POST/api/queue/allocate
Accepts { patientId, department, priorityTier } → queries MongoDB for all doctors in department → counts their active WAITING tickets → assigns to doctor with fewest tickets (shortest queue) → generates token (DEPT-NNN) → saves QueueTicket → returns full ticket object with estimated wait time.
